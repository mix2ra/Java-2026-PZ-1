# Відповіді на контрольні питання

## 1. Призначення JFC фреймворку і його архітектура

**Призначення JFC**
**JFC (Java Foundation Classes)** — це набір бібліотек Java для створення графічних інтерфейсів користувача (GUI).

JFC використовується для:
- створення віконних додатків;
- побудови кнопок, меню, текстових полів;
- обробки подій користувача;
- створення кросплатформених програм.

## Основні компоненти JFC
JFC включає:

- **AWT (Abstract Window Toolkit)**  
  Базова бібліотека для створення GUI.

- **Swing**  
  Розширена бібліотека GUI-компонентів.

- **Java 2D**  
  Для графіки, малювання, анімації.

- **Accessibility API**  
  Для підтримки доступності.

- **Drag and Drop API**  
  Для перетягування елементів.

## Архітектура JFC
Архітектура побудована за принципом:

**Компоненти → Контейнери → Менеджери розміщення → Обробка подій**

# 3. Приклад структури GUI програми на Java

## Приклад

```java
import javax.swing.*;

public class MyWindow extends JFrame {
    public MyWindow() {
        setTitle("Моя програма");
        setSize(400, 300);
        setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

        JPanel panel = new JPanel();

        JButton button = new JButton("Натисни");

        panel.add(button);

        add(panel);

        setVisible(true);
    }


    public static void main(String[] args) {
        new MyWindow();
    }
}
```

№7. Призначення об’єктів event і механізм оброблення

Події (event) - це об’єкти, які виникають у результаті дій користувача, наприклад натискання кнопки, введення тексту або руху миші. Пакет java.awt.event містить класи, що описують ці події. Механізм оброблення подій є подієво-орієнтованим. Спочатку користувач виконує дію, генерується подія, вона передається обробнику, після чого Listener виконує відповідний код.

---

№8. Призначення інтерфейсів типу XxxListener

Інтерфейси Listener використовуються для оброблення подій. Вони дозволяють програмі реагувати на дії користувача. Основні типи: ActionListener, MouseListener, KeyListener, WindowListener, FocusListener. Наприклад, ActionListener застосовується для оброблення натискання кнопки. Listener «слухає» подію і виконує потрібні дії у відповідь.
